# 第 2 週今日 Python 練習題：猜數字遊戲

## 題目說明

製作一個簡單的「猜數字遊戲」。程式隨機產生一個 1 到 100 之間的整數，讓使用者不斷猜測，直到猜中為止。每次猜錯要給出「太大」或「太小」的提示，並在猜中後顯示總共猜了幾次。

## 技術要求

- **概念**：`while` 迴圈、`break`、`continue`、`range`、`for` 迴圈（進階挑戰）
- **需要的模組**：`random`

## 具體任務

1. 使用 `random.randint(1, 100)` 產生一個隨機目標數字。
2. 用 `while True` 迴圈讓使用者持續輸入猜測值（記得用 `int()` 轉型）。
3. 判斷猜測值：
   - 若猜測值 **大於** 目標 → 印出「太大了！再猜一次：」
   - 若猜測值 **小於** 目標 → 印出「太小了！再猜一次：」
   - 若猜測值 **等於** 目標 → 印出「🎉 恭喜！你猜對了！共猜了 X 次。」並用 `break` 離開迴圈。
4. 每次猜測後，計數器加 1。
5. （選做）輸入非數字時不崩潰，改用 `continue` 跳過並提示重新輸入。

## 範例

```python
# 執行後可能的互動（目標數字為 42）

請猜一個 1 到 100 之間的數字：50
太大了！再猜一次：30
太小了！再猜一次：42
🎉 恭喜！你猜對了！共猜了 3 次。
```

## 提示

- `random.randint(a, b)` 會回傳 a 到 b **之間（含兩端）** 的隨機整數。
- `while True:` 會無限迴圈，必須靠 `break` 才能離開。
- 用一個變數（例如 `attempts = 0`）在每次猜測前加 1 來計數。
- `input()` 回傳的是字串，要用 `int(...)` 轉換才能做數字比較。

## 學習重點

| 概念 | 用途 |
|------|------|
| `while True` | 處理「不確定次數」的重複動作，爬蟲中常用來持續抓取直到沒有下一頁 |
| `break` | 滿足條件時立刻離開迴圈，爬蟲中可用來在找到目標資料後停止搜尋 |
| `continue` | 跳過這一輪，進入下一次迭代，常用來略過無效或空白的資料列 |
| `for` + `range` | 對已知次數或已知清單做固定次數的迭代，ML 訓練迴圈的基本骨架 |

## 進階挑戰（選做）

1. **限制次數**：只給使用者 7 次機會，超過後自動印出答案並結束遊戲。
2. **統計紀錄**：用 `for` 迴圈讓使用者連玩 3 輪，最後統計每輪猜了幾次，印出平均猜測次數。
3. **難度選擇**：遊戲開始前讓使用者選擇難度（簡單 1–50、普通 1–100、困難 1–500），用 `range` 或條件判斷決定隨機數的範圍。
