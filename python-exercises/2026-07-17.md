# 第 2 週今日 Python 練習題：猜數字遊戲

## 題目說明

你將實作一個「猜數字遊戲」。程式會隨機選定一個 1 到 100 之間的整數，玩家需要不斷猜測，程式則根據每次猜測給予「太大」、「太小」或「答對了！」的提示，直到猜中為止。

完成後，你將了解 `while` 迴圈、`break`、以及搭配條件判斷控制遊戲流程的方式——這些都是日後撰寫爬蟲「重試迴圈」與 ML「訓練迴圈」的基礎概念。

---

## 技術要求

- **概念**：`while`、`for`、`range`、`break`、`continue`、條件判斷
- **需要的模組**：`random`（Python 內建，不需安裝）

---

## 具體任務

1. 使用 `random.randint(1, 100)` 產生一個隨機目標數字，儲存在變數 `target` 中。
2. 初始化一個計數器 `attempts = 0`，用來記錄猜測次數。
3. 使用 `while True` 迴圈持續讓玩家輸入猜測（`input()`），並在迴圈內：
   - 將輸入轉為整數；若輸入非數字，用 `continue` 跳過並提示「請輸入數字」。
   - 每次有效猜測時 `attempts += 1`。
   - 若猜測 > target，印出「太大了！再小一點。」
   - 若猜測 < target，印出「太小了！再大一點。」
   - 若猜中，印出「🎉 答對了！你一共猜了 N 次。」，並用 `break` 結束迴圈。
4. 遊戲結束後，額外印出：「你一共猜了 N 次。平均每次的可能範圍縮小了 ... 倍。」（選做，見進階挑戰）

---

## 範例

```python
# 執行結果範例（假設 target = 42）
請猜一個 1 到 100 的整數：50
太大了！再小一點。
請猜一個 1 到 100 的整數：25
太小了！再大一點。
請猜一個 1 到 100 的整數：abc
請輸入整數數字！
請猜一個 1 到 100 的整數：37
太小了！再大一點。
請猜一個 1 到 100 的整數：42
🎉 答對了！你一共猜了 4 次。
```

---

## 提示

- `random.randint(a, b)` 包含兩端點，即 `a <= 結果 <= b`。
- `input()` 永遠回傳字串，記得用 `int()` 轉型。
- 用 `try / except ValueError` 捕捉非數字輸入，然後 `continue` 跳回迴圈頂端。
- `while True` 搭配 `break` 是 Python 中最常見的「等待直到成功」模式。

---

## 學習重點（跟爬蟲／ML 的關聯）

| 今天學的概念 | 爬蟲 / ML 的對應場景 |
|---|---|
| `while True` + `break` | 爬蟲重試迴圈：若請求失敗就重試，成功才 `break` |
| `continue` 跳過無效輸入 | 資料清理：跳過格式錯誤的列，繼續處理下一筆 |
| 計數器 `attempts` | ML 訓練的 `epoch` 計數 |
| 條件判斷縮小範圍 | 二分搜尋法（Binary Search）——許多最佳化演算法的核心 |

---

## 進階挑戰（選做）

1. **限制猜測次數**：加入 `max_attempts = 7` 的上限。若超過次數仍未猜中，印出「挑戰失敗！正確答案是 N。」並結束。（提示：用 `for i in range(max_attempts)` 取代 `while True`）
2. **難度選擇**：遊戲開始前讓玩家選擇難度（`easy=1~50`、`normal=1~100`、`hard=1~200`），並依難度設定 `random.randint` 的範圍。
3. **勝率統計**：用 `for` 迴圈讓玩家玩 5 局，記錄每局的猜測次數，最後用 `sum()` 與 `min()` 印出總次數與最佳成績。
