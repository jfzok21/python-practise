# 第 2 週今日 Python 練習題：猜數字遊戲

## 題目說明

設計一個「猜數字」互動小遊戲。程式會先隨機產生一個 1～100 之間的整數，玩家反覆輸入猜測值，程式給出「太大」、「太小」或「恭喜猜中！」的提示，直到猜對為止，最後顯示共猜了幾次。

## 技術要求
- 概念：`while` 迴圈、`break`、`if/elif/else`、型態轉換（`int()`）、`random` 模組
- 需要的模組：`random`

## 具體任務

1. 使用 `random.randint(1, 100)` 產生目標數字，並初始化猜測次數計數器為 `0`。
2. 進入 `while True` 迴圈，每次迴圈：
   - 用 `input()` 接收玩家輸入，轉成整數後計數器加 `1`。
   - 判斷輸入值與目標的大小關係，印出對應提示。
   - 若猜中，印出恭喜訊息與總次數後用 `break` 跳出迴圈。
3. （選做）若輸入非數字，用 `try/except` 捕捉例外並提示重新輸入，**不**增加計數。

## 範例

```python
# 執行範例（目標數字假設為 42）
請猜一個 1～100 之間的整數：70
太大了，再試試看！
請猜一個 1～100 之間的整數：30
太小了，再試試看！
請猜一個 1～100 之間的整數：42
🎉 恭喜！你猜對了，答案就是 42，共猜了 3 次！
```

## 提示
- `while True:` 搭配 `break` 是「直到滿足條件才停止」的常用模式。
- `random.randint(a, b)` 會回傳包含兩端點的整數。
- 記得在迴圈開始前把計數器設成 `0`，每次猜測時才能正確累加。
- 若想挑戰選做任務，`try: int(user_input) except ValueError:` 可攔截非數字輸入。

## 學習重點

| 概念 | 說明 | 爬蟲 / ML 的關聯 |
|------|------|----------------|
| `while` 迴圈 | 重複執行直到條件為假 | 爬蟲：持續翻頁直到沒有「下一頁」 |
| `break` | 立即跳出迴圈 | 找到目標資料後立即停止爬取，節省時間 |
| 計數器模式 | 追蹤已執行的次數 | ML：記錄訓練 epoch 數或已爬取頁數 |
| `try/except` | 例外處理 | 爬蟲：網路超時或解析失敗時不中斷程式 |

## 進階挑戰（選做）

1. **限制次數**：給玩家最多 7 次機會，超過後印出「很遺憾，答案是 XX」並結束，使用 `for i in range(7)` 改寫迴圈。
2. **難易度選擇**：遊戲開始時讓玩家選擇「簡單（1-50）」或「困難（1-200）」，依選擇調整 `randint` 的範圍。
3. **連續遊戲**：猜完一局後詢問「是否再玩一局？(y/n)」，利用外層 `while` 迴圈控制重複。
